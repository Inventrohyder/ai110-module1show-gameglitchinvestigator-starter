# GitHub Labels and Board Configuration

To manage and track the progress of project milestones, we have configured custom issue labels and a project board:

## Custom Labels

- `category:core` (color: `0072F0`) - Core project milestones and requirements.
- `category:stretch` (color: `8E44AD`) - Stretch challenges and advanced features.
- `category:infra` (color: `E67E22`) - Infrastructure, setups, linters, testing.
- `phase:setup` (color: `F1C40F`) - Initial scaffolding and dev setup.
- `phase:discovery` (color: `E74C3C`) - Glitch hunting and verification recording.
- `phase:repair` (color: `2ECC71`) - Bug repair and code fixes.
- `phase:stretch` (color: `9B59B6`) - Stretch implementation phase.
- `priority:p0` (color: `D35400`) - Blocking / Critical.
- `priority:p1` (color: `2980B9`) - High priority / Required.
- `priority:p2` (color: `7F8C8D`) - Medium/Low priority / Stretch.

## Project Board

We created a GitHub Project board titled **Game Glitch Investigator** to track backlog items and assign labels, categories, priorities, and project status columns.
