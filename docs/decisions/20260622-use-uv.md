# Use uv for Dependency Management

- Status: accepted
- Deciders: Antigravity, USER
- Date: 2026-06-22

## Context and Problem Statement

The starter repository uses standard `requirements.txt` which lacks:

- Strict dependency locking (leading to "works on my machine" issues).
- Fast virtual environment creation and package installation.
- Standardized python version enforcement.
- Streamlined dependency upgrades.

We need a modern, fast, and secure Python package manager.

## Decision Drivers

- High speed of package installations.
- Lockfile support for reproducible builds.
- Python toolchain management (ensuring correct Python version like 3.14).
- Standardized `pyproject.toml` metadata structure.

## Considered Options

1. **pip + requirements.txt**: Traditional approach with no lockfile by default (unless using pip-compile).
2. **Poetry**: Modern, full-featured package manager, but can be slow.
3. **uv**: Extremely fast, rust-based dependency resolver and installer with native lockfile and python toolchain management support.

## Decision Outcome

Chosen option: **uv** (Option 3), because it is significantly faster than pip and poetry, supports lockfile-based reproduction, handles multi-version Python installations seamlessly, and conforms to PEP 621.

### Positive Consequences

- Lockfile-based reproducible environments (`uv.lock`).
- Sub-second package resolution.
- Automatic Python version management and bootstrapping.

### Negative Consequences

- Developers need to install `uv` CLI.
