## 🕵️‍♂️ Your Mission & Core Findings

### Game Purpose

The application is an interactive, web-based number guessing game built using Streamlit. The objective is for a player to guess a randomly generated hidden secret integer within a specified boundary range before running out of allotted attempts.

### Bugs Identified & Fixes Applied

1. **Bug 1: Inverted Hint Messages**

   - **Root Cause:** The comparison conditional block in `check_guess` had its emoji indicators and instructional string literals completely reversed. A guess higher than the secret returned `"📈 Go HIGHER!"`, causing misleading player loops.
   - **Fix Applied:** Modified `check_guess` in `app.py` to properly match directional hints with mathematical bounds, returning `"📉 Go LOWER!"` when a guess exceeds the secret target.

2. **Bug 2: Type-Juggling on Even Attempts**

   - **Root Cause:** On every even-numbered attempt, the runtime cast the secret key into a string primitive (`str()`). When evaluated against an integer guess, this triggered a `TypeError` fallback to alphanumeric lexicographical string parsing, leading to hidden evaluation failures (e.g., evaluating `"6" > "50"` as `True`).
   - **Fix Applied:** Removed the conditional type-casting logic block entirely, ensuring the target secret number remains a stable Python integer cross-compared numerically on every single attempt.

3. **Bug 3: Hard Difficulty Range Inverted**

   - **Root Cause:** The range function for "Hard" mode was hardcoded to return a boundary limits tuple of `1, 50`, which was significantly narrower and easier than "Normal" difficulty bounds (`1, 100`).
   - **Fix Applied:** Adjusted the range progressively to return `1, 150` for "Hard" mode, making the game bounds scale properly alongside the reduced attempt limitations.

## 🚀 Stretch Features

### Challenge 2: Proximity Distance Tracking History Line Chart

- **Implementation Details:** Integrated a non-disruptive session tracking array structure inside `st.session_state.distances`. When a user enters a valid numeric entry guess, the system automatically computes the absolute numerical differential distance offset ($|\text{guess} - \text{secret}|$) and appends the scalar metric to the historical tracking matrix.
- **UI Interface Rendering:** If elements are recorded inside the distance array, the layout dynamically exposes a sidebar split component using `st.sidebar.line_chart` to visually present the user's accuracy over successive attempts.
